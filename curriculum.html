12주 커리큘럼

판단하고 → 만들고 → 실행한다 — 실행형 커리큘럼
총 12회 주 1회 2시간 30분 저녁 19:00~21:30
BASIC Phase 1 — AI 리더의 눈 1~4주
Week 1 AI 시대, CEO의 판단력
생성형 AI의 큰 그림을 이해하고, CEO가 가져야 할 판단 프레임워크를 익힙니다.

학습 내용

  • 생성형 AI → 에이전트 → 바이브코딩 3가지 변화
  • CEO의 "구축/구매/무시" 의사결정 프레임워크
  • AI가 바꾸는 비즈니스 모델

산출물

  • ChatGPT·Claude로 경영 시나리오 3가지 체험
  • AI 기회 캔버스 초안 (템플릿)
Week 2 AI Native 기업은 이렇게 일한다
AI를 도입해 일하는 방식 자체를 바꾼 기업들의 사례를 봅니다. 네트워킹 주차.

학습 내용

  • 국내 AI Native 기업 사례 (뤼튼, 업스테이지, 채널톡, 당근)
  • 뽀짝이(GPTers) AI 에이전트 운영 사례
  • "도구를 아는 것" vs "일하는 방식을 바꾸는 것"

산출물

  • 업종별 테이블 토론 + 네트워킹
  • 캔버스 보강·수정
Week 3 AI의 가능성과 한계 + 활용 가이드
AI 리스크를 알고 안심한 뒤, 내일부터 업무에 AI를 활용하는 방법을 배웁니다.

학습 내용

  • 데이터 보안·개인정보·저작권·환각·AI 규제 동향
  • AI 활용 단계별 가이드 — 프롬프트부터 조직 확산까지
  • 프롬프트 설계 원칙 (역할, 맥락, 형식, 제약)

산출물

  • 업무 후보별 리스크·실현가능성 평가표
  • Top 3 1차 선정
  • "이번 주 AI로 해볼 업무 1개" 메모
Week 4 바이브코딩 첫 체험 + 실행 계획
"말로 설명하면 앱이 된다"를 직접 체험하고, 향후 8주 실행 계획을 세웁니다.

학습 내용

  • 바이브코딩이란? — 코드를 몰라도 앱을 만드는 시대
  • Google AI Studio 소개 — 가장 쉬운 시작점

산출물

  • Google AI Studio 바이브코딩 실습 (90분)
  • Top 3 최종 확정 + 실행 계획서
INTERMEDIATE Phase 2 — 도구와 실전 적용 5~8주
Week 5 Cowork + MCP — 데스크탑 바이브코딩
Claude 데스크탑앱으로 본격적인 바이브코딩을 체험하고, MCP로 외부 도구와 연결합니다.

학습 내용

  • Cowork + MCP 소개 — Google AI Studio보다 한 단계 위
  • MCP: AI가 외부 도구(Slack, 캘린더, DB 등)와 연결
  • 프롬프트를 넘어 업무 도구를 직접 만드는 단계

산출물

  • Cowork + MCP + AI 실습 플랫폼 (90분 실습)
  • 프로토타입 v1 데모
Week 6 Claude Code + 에이전트
CLI 기반의 가장 강력한 바이브코딩 도구와, "시키면 알아서 하는" 에이전트를 체험합니다.

학습 내용

  • Claude Code란? Cowork과 뭐가 다른가 — 데모 중심
  • 에이전트: "지시하면 알아서 실행하는 AI"
  • Cowork vs Claude Code: 각각 언제 쓰는가

산출물

  • Claude Code + AI 실습 플랫폼 체험 (60분)
  • 프로토타입 업그레이드 계획서
Week 7 조직 자산화 + ROI
한 사람이 만든 노하우가 조직 전체의 자산이 되는 구조와, 투자 대비 효과를 측정합니다.

학습 내용

  • AI 업무 자동화를 조직 공유 자산으로 만드는 방법
  • 조직 확산 구조: 한 사람의 경험이 팀 전체의 역량이 된다
  • ROI 산출법: 비용 구조 비교

산출물

  • Skill 설계 + ROI 산출 워크숍 (60분)
  • Skill Top 5 설계서 + ROI 표
Week 8 중간 현황 발표
1개월간 조직에서 AI를 어떻게 활용했는지 공유합니다.

학습 내용

  • 발표 구조: "계획한 것 → 시도한 것 → 결과 → 배운 것"
  • 3가지 카테고리: Pioneer / Explorer / Planner

산출물

  • 그룹 내 발표 → Best-of 쇼케이스
  • 전문가 피드백 + Phase 3 개선 목표
ADVANCED Phase 3 — 완성과 확산 9~12주
Week 9 프로토타입 개선 + 게스트 CEO
피드백을 반영하여 프로토타입을 개선하고, 성공 사례로 검증합니다.

학습 내용

  • 프로토타입 개선 전략 — "쓸 수 있는 것 vs 보여주기용"
  • 게스트 CEO 특강 — AI 도입 성공 사례

산출물

  • Claude Code + AI 실습 플랫폼에서 개선 (60분 실습)
  • 개선 프로토타입 + 전파 실험 계획
Week 10 조직 전파 — 회고와 전략
이론이 아닌 경험에서 출발합니다. 전파의 성공/실패를 돌아보고 전략을 세웁니다.

학습 내용

  • AI 전담팀 vs 전사 내재화 — 우리에게 맞는 모델
  • 직원 저항 극복 — 변화 관리 전략
  • Skill 공유 체계 + AI 실습 플랫폼 활용

산출물

  • 전파 회고 + 로드맵 설계 워크숍 (60분)
  • 전파 로드맵 v2 (3/6/12개월)
Week 11 발표 리허설 + 피어 리뷰
CEO끼리 서로의 AI 전환 계획을 리뷰하고 보강합니다.

학습 내용

  • 좋은 AI 전환 발표의 조건
  • 5분 안에 설득력 있게 전달하는 법

산출물

  • 피어 리뷰 워크숍 (70분)
  • 피어 피드백 반영 발표 자료 완성
Week 12 최종 발표 + 수료
3개월간의 AI 전환 여정을 정리하고, 실제 결과를 발표합니다.

학습 내용

  • AI 도입 배경과 목표
  • 시도한 것과 결과
  • 조직 전파 현황과 계획
  • ROI 분석 + 향후 로드맵

산출물

  • 그룹 내 발표 → Best-of 쇼케이스
  • 시상: "최고 전환 여정" / "가장 용감한 피봇" / "최고 조직 확산"
  • 수료식 + 네트워킹 디너
